• 启动与陷入

  - kernel/entry.S：内核最早执行的入口汇编，负责给每个 hart 准备初始栈并跳转到 start()，它解决的是“CPU 刚开始跑时 C 环境还不存在”的问题。
  - kernel/start.c：在 machine mode 下完成最初的硬件准备，设置委托、中断、计时器和 hartid，然后切到 supervisor mode 并进入 main()。
  - kernel/main.c：内核主初始化流程，CPU0 负责初始化控制台、内存、页表、进程表、文件系统、磁盘和中断等全局子系统，其他 CPU 做各自局部初始化后进入调度器。
  - kernel/kernel.ld：链接脚本，规定内核各段的虚拟地址布局、入口位置以及符号如 etext/end 的生成，保证镜像按 xv6 预期方式装入内存。
  - kernel/kernelvec.S：内核态 trap 入口，负责在 trap 发生时保存寄存器、调用 kerneltrap()，返回前再恢复现场。
  - kernel/trampoline.S：用户态与内核态切换用的“跳板页”代码，里面的 uservec 和 userret 负责用户 trap 进入内核以及从内核安全返回用户态。
  - kernel/trap.c：trap 处理核心逻辑，区分系统调用、设备中断和异常，协调 usertrap()、kerneltrap()、usertrapret() 以及时钟 tick 等行为。
  - kernel/swtch.S：最底层上下文切换代码，只保存/恢复内核线程上下文，不碰用户寄存器，是调度器切换进程时的关键汇编例程。
  - kernel/riscv.h：RISC-V 架构相关头文件，封装 CSR 读写、页表位、地址转换、屏障和中断控制等底层宏/内联函数，几乎所有内核核心代码都会依赖它。

  进程、调度与内存

  - kernel/proc.h：定义进程、CPU、本地上下文、trapframe 等核心数据结构，是进程调度和 trap 返回路径共享的“结构骨架”。
  - kernel/proc.c：实现进程创建、退出、等待、睡眠/唤醒、调度和上下文切换等完整生命周期逻辑，是 xv6 进程管理的中心文件。
  - kernel/kalloc.c：页粒度物理内存分配器，维护空闲页链表，向页表、进程、管道和文件系统等模块提供最基础的物理页。
  - kernel/vm.c：虚拟内存管理核心，负责建立内核页表、创建/复制/销毁用户页表、地址映射、用户内存拷贝以及用户地址合法性检查。
  - kernel/vm.h：这个仓库里用于声明虚拟内存相关额外策略常量的头文件，目前可见是给 sbrk 分配模式之类扩展留接口。
  - kernel/memlayout.h：定义 xv6 的物理/虚拟地址布局，包括 KERNBASE、PHYSTOP、TRAMPOLINE、TRAPFRAME、设备 MMIO 地址和内核栈布局。
  - kernel/spinlock.h：定义自旋锁结构体，描述锁名、持有状态和持有 CPU 等信息，是所有短临界区同步的基础。
  - kernel/spinlock.c：实现自旋锁的获取、释放以及中断屏蔽配合逻辑，保证多核下内核共享数据结构的并发安全。
  - kernel/sleeplock.h：定义睡眠锁结构，用于可能阻塞较久的临界区，典型场景是 inode 和磁盘块等文件系统对象。
  - kernel/sleeplock.c：实现睡眠锁，获取不到锁时进程会睡眠而不是忙等，适合长时间持锁的路径。
  - kernel/param.h：集中定义系统规模参数，例如最大进程数、CPU 数、文件数、inode 数、缓冲块数和文件系统大小等。
  - kernel/types.h：提供整个内核通用的基础类型别名，如 uint、ushort、pde_t 一类，让各模块共享统一类型定义。

  系统调用与执行

  - kernel/syscall.h：定义系统调用号，把用户态请求和内核态分发索引对应起来。
  - kernel/syscall.c：系统调用分发器，负责从 trapframe 取参数和调用号，调用对应 sys_* 处理函数，并把返回值写回用户寄存器。
  - kernel/sysproc.c：实现与进程/时间/地址空间直接相关的系统调用，如 fork、exit、wait、sbrk、sleep/pause、kill、uptime 等。
  - kernel/sysfile.c：实现与文件、目录和文件描述符相关的系统调用，如 open、read、write、close、link、unlink、mkdir 等。
  - kernel/exec.c：实现 exec，把 ELF 可执行文件装入新页表、建立用户栈、拷贝参数并替换当前进程映像，是“同一进程换程序”的关键路径。
  - kernel/elf.h：定义 ELF 文件头和程序头格式，供 exec.c 按标准解析可执行文件。

  文件系统与文件对象

  - kernel/fs.h：定义磁盘文件系统的核心磁盘格式，包括 superblock、dinode、目录项、位图和块尺寸等，是 mkfs 和内核共享的协议头。
  - kernel/file.h：定义内核中的 file、inode、设备开关表 devsw 等抽象，把“打开文件”和“磁盘 inode”这两层对象区分开。
  - kernel/file.c：实现全局文件表、文件引用计数、read/write/stat/close/dup 等统一文件接口，是系统调用层和底层 inode/pipe/设备之间的桥梁。
  - kernel/fs.c：实现 inode 层和路径名层的主要逻辑，包括块映射、inode 分配回收、目录查找、路径解析以及文件内容读写。
  - kernel/log.c：实现简单的写前日志文件系统，保证多个块更新在崩溃后仍可恢复一致，是 xv6 文件系统“事务性”的来源。
  - kernel/buf.h：定义磁盘缓冲块 struct buf，描述块号、设备号、有效位、脏位、锁和数据区等字段。
  - kernel/bio.c：实现块缓存层，负责缓存磁盘块、维护 LRU 风格链表、协调 bread/bwrite/brelse，减少磁盘访问并串联日志层与驱动层。
  - kernel/pipe.c：实现匿名管道，提供一个简单的内核环形缓冲区和同步逻辑，让进程间能按文件接口进行字节流通信。
  - kernel/stat.h：定义 stat 结构和文件类型常量，供内核与用户态描述文件元信息。
  - kernel/fcntl.h：定义 open 使用的标志位，如只读、只写、读写、创建和截断，是系统调用参数约定的一部分。

  设备与基础库

  - kernel/console.c：实现控制台设备的高层逻辑，负责行缓冲输入、退格和控制字符处理，并把读写统一接到字符设备接口上。
  - kernel/uart.c：直接驱动 UART 串口硬件，负责初始化、发送、接收和中断配合，是 console 最底层的字符 I/O 实现。
  - kernel/printf.c：提供内核态格式化输出和 panic 机制，调试、报错和崩溃时的输出都从这里走。
  - kernel/plic.c：配置和操作 RISC-V 平台级中断控制器 PLIC，负责打开设备中断、认领中断和完成中断。
  - kernel/virtio.h：定义 VirtIO MMIO 和块设备队列描述符格式，是磁盘驱动和虚拟设备通信时必须遵守的硬件协议头。
  - kernel/virtio_disk.c：实现 VirtIO 块设备驱动，负责初始化 virtqueue、提交读写请求、等待完成并处理中断，是 xv6 的磁盘 I/O 后端。
  - kernel/string.c：提供内核自用的 memset、memmove、memcmp、strlen 等基础字符串/内存函数，替代标准库。
  - kernel/defs.h：整个内核的总前向声明和函数原型汇总，相当于一个跨模块“公共接口清单”，减少循环包含。
  - kernel/buf.h：虽然属于文件系统，但它也承担通用缓存对象定义的作用，供 bio.c、log.c、fs.c 等多个模块共享。
  - kernel/file.h：虽然属于文件系统，但同时也是设备、管道、inode 统一抽象的接口层，因此系统调用和驱动层都会依赖它。
